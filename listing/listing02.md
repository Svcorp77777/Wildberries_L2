Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
2
1

Этот ответ был получен из-за особенностей работы defer.
Так как в данном примере отложенная функция является функциональным литералом,
а окружающая функция имеет именованные параметры результата, которые находятся в 
области видимости внутри литерала, отложенная функция может получить доступ 
к параметрам результата и изменить их до того, как они будут возвращены.

При выполнении функций переменная х будет изменена в обоих случаях, но существует 
разница.

При выполнении функции test() х равно 1-му, и происходит return на этом этапе вызывается 
defer и так как возвращается именованная переменная defer будет работать именно с ней
и соответственно х будет увеличено на единицу.

При выполнении функции anotherTest() х является локальной переменной и равно 1-му и кладется
в return, после чего вызывается defer и прибавляет единицу но это не влияет х который был
отправлен в return, поэтому и был получен такой результат.
```
